# Security Policy

## Supported versions

The latest release on the default branch is supported. Security fixes are
released for the most recent tagged version.

| Version | Supported |
| --- | --- |
| Latest | Yes |
| Older tags | Best effort |

## Reporting a vulnerability

Please do not open public issues for security vulnerabilities.

Report vulnerabilities privately by emailing:
- opensource@token-saver.dev

Include the following details when possible:
- A clear description of the issue and impact
- Steps to reproduce
- A minimal proof of concept
- Affected version, OS, and shell

We will acknowledge reports within 3 business days and provide status updates
until a fix or mitigation is available.

## Disclosure process

- We validate the report and assess severity.
- We prepare and test a fix.
- We publish a release and advisory notes.
- We coordinate public disclosure with the reporter.
